セットアップ

s2dao.php5.version.tar.gz をダウンロード後、解凍してください。以下のようなディレクトリが展開されます。

s2dao.php5/
  +-- docs/                     ----- ドキュメント ディレクトリ
  +-- src/                      ----- ソース ディレクトリ
  +-- s2dao.inc.php             ----- S2Dao.PHP5 設定ファイル
  +-- Apache_Software_License_2.0.txt
  +-- ChangeLog

S2Container.PHP5が必要です。
S2Container.PHP5のセットアップを終えてからS2Dao.PHP5の設定をしてください。
また、S2Dao.PHP5ではPDOを利用しています。 利用前にPDOのインストールも終えておいてください。

S2Dao.PHP5-1.0.0-beta2の設定

通常 s2dao.inc.php を読み込むだけでS2Dao.PHP5の利用が可能です。

s2dao.inc.php の設定は以下の通りです。

/** S2Dao.PHP5 ROOT Directory; 定数S2DAO_PHP5にはS2Dao.PHP5の設置してあるディレクトリを指定してください。 */
define("S2DAO_PHP5", dirname(__FILE__) . "/src/s2dao.php5");

/** PDO DataSource dicon; 定数PDO_DICON には pdo.diconへのパスを指定してください(任意) */
//define("PDO_DICON", dirname(__FILE__) . "/pdo.dicon");

/** S2Dao.PHP5 ClassLoader */
require_once(S2DAO_PHP5 . "/S2DaoClassLoader.class.php");

if( class_exists("S2ContainerClassLoader") ){
    S2ContainerClassLoader::import(S2DaoClassLoader::export());
}
if( class_exists("S2Container_MessageUtil") ){
    S2ContainerMessageUtil::addMessageResource(S2DAO_PHP5."/DaoMessages.properties");
}

詳細は src/example/example.inc.php に記述しているので参考にしてください。

※S2Dao.PHP5-1.0.0-beta1の時のように __autoload() 関数内にS2DaoClassLoaderを設定することはありません。

DICONの設定

S2Dao.PHP5を利用するには src/s2dao.php5/dao.dicon を読み込んでください。
事前にpdo.diconの設定をしておいてください。

dao.diconは以下のように記述されています。

<components namespace="dao">

    <include path="%PDO_DICON%" />

    <component class="S2Dao_BasicResultSetFactory" />
    <component class="S2Dao_BasicStatementFactory" />

    <component class="S2Dao_DaoMetaDataFactoryImpl" />
    <component name="interceptor" class="S2DaoInterceptor" />

</components>

また、読み込むべきdiconの例です。

<components namespace="example">

    <include path="%S2DAO_PHP5%/dao.dicon" />

    <component name="beanCdDao" class="Cd1Dao">
        <aspect>dao.interceptor</aspect>
    </component>

    <component name="daoImpl" class="Cd2DaoImpl">
        <aspect>dao.interceptor</aspect>
    </component>

</components>

詳細は src/example/example.dicon.xml に記述しているので参考にしてください。

Dao,Entityの読み込みの例(S2ContainerClassLoader::importを用いて)

src/example/example.inc.php では 以下のように記述し、複数個生成された dao 及び entity(bean)の読み込みを自動化しています。

if( class_exists("S2ContainerClassLoader") ){
    S2ContainerClassLoader::import(dirname(__FILE__) . "/dao");
    S2ContainerClassLoader::import(dirname(__FILE__) . "/entity");
    S2ContainerClassLoader::import(dirname(__FILE__) . "/impl");

    function __autoload($class = null){
        if( S2ContainerClassLoader::load($class) ){
            return;
        }
    }
}

詳細は src/example/example.inc.php に記述しているので参考にしてください。

PDOの設定

S2Dao.PHP5-1.0.0-beta2からは PDO を利用しています。PDO で利用できる DSN については PHP: PDO Functions - Manualを参考に設定してください。

pdo.diconファイルを読み込ませるには、定数 PDO_DICON に pdo.diconまでのパスを指定してください。

define("PDO_DICON", dirname(__FILE__) . "/pdo.dicon");

また、 src/example/pdo.dicon には以下のように記述されています。

<components namespace="pdo">

    <component name="dataSource" class="S2Container_PDODataSource">
        <property name="dsn">"mysql:host=localhost; dbname=s2con"</property>
        <property name="user">"root"</property>
        <property name="password">"pass"</property>
    </component>

    <!--
    <component name="dataSource" class="S2Container_PDODataSource">
        <property name="dsn">"sqlite:/path/to/s2con.db"</property>
    </component>
    -->

    <!--
    <component name="dataSource" class="S2Container_PDODataSource">
        <property name="dsn">"pgsql:host=localhost dbname=s2con"</property>
        <property name="user">"root"</property>
        <property name="password">"pass"</property>
    </component>
    -->

    <component name="dbtx" class="S2Container_PDOTxInterceptor" />
    <component name="session" class="S2Container_PDOSessionImpl" />

</components>

詳細は src/example/example.inc.php 及び src/example/example.dicon.xml を参考にしてください。

※S2Dao.PHPP5-1.0.0-beta1まで利用していた PEAR::DB は利用でませんのでご注意ください。

S2Dao.PHP5-1.0.0-beta1の設定

※S2Dao.PHP5-1.0.0-beta1を利用する場合はS2Container.PHP5のバージョンがS2Container.PHP5-1.0.0-beta2に限定されます。

s2dao.inc.phpを適切に設定し、 s2dao.inc.php を読み込むことで S2Dao.PHP5を利用することができます。

require_once "/path/to/s2dao.inc.php";

s2dao.inc.php での設定

<?php

/** S2Container.PHP5 を設置したディレクトリを設定します。 */
define('S2CONTAINER_PHP5', dirname(__FILE__) . "/src/s2container.php5");
/** S2Dao.PHP5 が設置されているディレクトリを設定します。 */
define("S2DAO_PHP5", dirname(__FILE__) . "/src/s2dao.php5");

/** DICON XML format DTD Validation Switch */
define("S2CONTAINER_PHP5_DOM_VALIDATE", false);

/**
 * Messages Resouce File
 * S2Container.php5 と S2Dao.php5 のメッセージをマージしたものです。
 */
define("S2CONTAINER_PHP5_MESSAGES_INI", S2DAO_PHP5 . "/SSRMessages.properties");

/** S2Container.PHP5 Log Level */
//define('S2CONTAINER_PHP5_LOG_LEVEL',SimpleLogger::WARN);

/** SingletonS2ContainerFactory app.dicon */
//define('S2CONTAINER_PHP5_APP_DICON','app.dicon');

require_once(S2CONTAINER_PHP5 . "/S2ContainerClassLoader.class.php");
require_once(S2CONTAINER_PHP5 . "/s2container.core.classes.php");
require_once(S2DAO_PHP5 . "/S2DaoClassLoader.class.php");

/** PEAR::DBが必須です。 */
require_once("DB.php");

if( function_exists("__autoload") ){
    exit("sorry. is already __autoload()");
}

// when beta2
function unsetS2ContainerClsLoderKey($key){
    if(isset(S2ContainerClassLoader::$CLASSES[$key])){
        unset(S2ContainerClassLoader::$CLASSES[$key]);
    }
}

/** __autoload function */
if( class_exists("S2ContainerClassLoader") && class_exists("S2DaoClassLoader") ){

    // unset
    unsetS2ContainerClsLoderKey("DaoMetaData");
    unsetS2ContainerClsLoderKey("DaoMetaDataImpl");
    unsetS2ContainerClsLoderKey("DaoMetaDataFactory");
    unsetS2ContainerClsLoderKey("DaoMetaDataFactoryImpl");
    unsetS2ContainerClsLoderKey("SqlCommand");

    function __autoload($class = null){
        if( S2ContainerClassLoader::load($class) ){
            return;
        }
        if( S2DaoClassLoader::load($class) ){
            return;
        }
    }
}
?>
__autoload()について

S2Dao.PHP5-1.0.0-beta1ではS2Container.PHP5 同様にクラス読み込み時に __autoload() を利用しています。
s2dao.inc.php で定義されている __autoload() は S2Container.PHP5 のクラスも読み込みますので、 S2Container.PHP5 で別途 __autoload() が定義されている場合はコメントアウトしてください。

<?php
:
:
/*
function __autoload($class=null){
    if(S2ContainerClassLoader::load($class)){return;}
}
*/
:
:
?>

他のフレームワークで __autoload() が定義されている場合は S2Container.PHP5のセットアップと同じ手順を踏んでください。

S2Container.PHP5-1.0.0-beta1で除外されるクラス

S2Container.PHP5 の beta2 では一部のクラス(extension/db以下)が S2Dao.PHP5 と重複することで予期せぬ動作をするので S2ContainerClassLoader 上から一部のクラスファイルを読み込まないようにしています。
以下にS2Container.PHP5(beta2)から読み込みを行わないクラスを列挙します。

  • DaoMetaData
  • DaoMetaDataImpl
  • DaoMetaDataFactory
  • DaoMetaDataFactoryImpl
  • SqlCommand
